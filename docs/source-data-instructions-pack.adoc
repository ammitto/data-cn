# Ammitto Source Data Instructions Pack

:toc: macro
:toclevels: 3
:source-highlighter: rouge
:icons: font

This document provides comprehensive instructions for adding a new sanctions data source to the Ammitto ecosystem.

== Overview

The Ammitto sanctions data system follows the **Open/Closed Principle**:

* **Open for Extension**: New sources can be added without modifying existing code
* **Closed for Modification**: The core harmonization engine remains unchanged

=== Repository Architecture

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        AMMITTO ECOSYSTEM                                     │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────┐     ┌─────────────────────┐     ┌─────────────────────┐
│   data-{source}     │     │      ammitto        │     │  ammitto.github.io  │
│   (Data Repo)       │     │      (Gem)          │     │     (Website)       │
├─────────────────────┤     ├─────────────────────┤     ├─────────────────────┤
│ - YAML source files │     │ - Source parsers    │     │ - Vue components    │
│ - JSON schemas      │     │ - Transformers      │     │ - API endpoints     │
```

== Step 1: Create Data Repository

Create a new Git repository named `data-{source}` where `{source}` is the lowercase source code (e.g., `data-uk`, `data-eu`, `data-jp`).

=== Directory Structure

```
├── sources/
│   ├── sanction-lists/           # Main sanction list data
│   │   ├── {list-type-1}/        # e.g., "consolidated-list", "asset-freeze"
│   │   │   ├── {entry-id}.yml    # Individual sanction entries
│   │   │   └── ...
│   │   └── {list-type-2}/
```

== Step 2: Define Source Schemas

Create JSON Schema files that validate your source-specific YAML structure.

=== Example: Sanction Announcement Schema

[source,yaml]
----
# schemas/{source}-announcement.yml
---
$schema: https://json-schema.org/draft/2020-12/schema
$id: https://www.ammitto.org/schemas/{source}-announcement.yml
title: Sanction Announcement
description: Schema for sanction announcement from {source}
type: object
required:
  - id
  - announcement
  - sanction_details
properties:
  id:
    type: string
    description: Unique identifier for this announcement
    pattern: "^{source}/[a-z0-9-]+$"

  announcement:
    type: object
    required:
      - title
      - publish_date
    properties:
      title:
        type: string
        description: Title of the announcement
      url:
        type: string
        format: uri
        description: URL to official source
      publish_date:
        type: string
        format: date
        description: Publication date (YYYY-MM-DD)
      publish_time:
        type: string
        pattern: "^[0-2][0-9]:[0-5][0-9]$"
        description: Publication time (HH:MM)
      document_id:
        type: string
        description: Official document identifier
      document_type:
        type: string
        description: Type of document (links to document-types.yml)
      authority:
        type: string
        description: Issuing authority (links to organizations.yml)
      publisher:
        type: string
        description: Publisher (links to organizations.yml)
      signatory:
        type: string
        description: Signatory (links to organizations.yml)
      content:
        type: string
        description: Full text content
      lang:
        type: string
        pattern: "^[a-z]{2,3}(-[A-Za-z]{4})?(-[A-Z]{2})?$"
        description: Language code (e.g., "zh-Hans", "en")
        default: en

  sanction_details:
    type: array
    items:
      type: object
      required:
        - entity
        - measures
      properties:
        entity:
          type: object
          required:
            - type
            - name
          properties:
            type:
              type: string
              enum: [person, organization, vessel, aircraft]
            name:
              type: object
              properties:
                full_name:
                  type: string
                first_name:
                  type: string
                last_name:
                  type: string
                script:
                  type: string
                  description: ISO 15924 script code
            aliases:
              type: array
              items:
                type: string
            identifiers:
              type: array
              items:
                type: object
                properties:
                  type:
                    type: string
                  value:
                    type: string
                  issuing_country:
                    type: string
        measures:
          type: array
          items:
            type: object
            properties:
              type:
                type: string
                enum:
                  - asset_freeze
                  - travel_ban
                  - arms_embargo
                  - trade_restriction
                  - import_ban
                  - export_ban
                  - investment_ban
                  - sectoral_sanction
                  - other
              description:
                type: object
                description: Localized description
        reasons:
          type: array
          items:
            type: object
            properties:
              category:
                type: string
              description:
                type: object
        period:
          type: object
          properties:
            effective_date:
              type: string
              format: date
            expiry_date:
              type: string
              format: date
            is_indefinite:
              type: boolean

  legal_citations:
    type: array
    description: Legal instruments cited as basis
    items:
      type: object
      properties:
        legal_instrument_id:
          type: string
          description: Reference to legal instrument
        articles:
          type: array
          items:
            type: string
        citation_type:
          type: string
          enum: [legal_basis, reference, amendment]
----

== Step 4: Create Ammitto Gem Source Module

In the `ammitto` gem, create a new source module.

=== Transformer Implementation

[source,ruby]
----
# ammitto/lib/ammitto/sources/{source}/transformer.rb
# frozen_string_literal: true

require_relative '../../transformers/base_transformer'

module Ammitto
  module Sources
    module NewSource
      # Transformer converts source-specific models to harmonized ontology
      #
      # @example
      #   transformer = Ammitto::Sources::NewSource::Transformer.new
      #   result = transformer.transform(source_model)
      #   entity = result[:entity]
      #   entry = result[:entry]
      #
      class Transformer < Ammitto::Transformers::BaseTransformer
        # Map source-specific regime names to codes
        REGIME_CODES = {
          'Regime A' => 'REGIME_A',
          'Regime B' => 'REGIME_B'
        }.freeze

        def initialize
          super(:new_source)  # Pass source code to base
        end

        # Transform source announcement to harmonized models
        # @param announcement [Announcement] source announcement model
        # @return [Hash] { entities:, entries:, group:, official_announcement: }
        def transform_announcement(announcement)
          entities = []
          entries = []

          announcement.sanction_details.each do |detail|
            entity = create_entity(detail.entity)
            entry = create_entry(detail, entity.id)

            entity.add_sanction_entry(entry)
            entities << entity
            entries << entry
          end

          group = create_group(announcement, entries)
          official_announcement = create_official_announcement(announcement)

          {
            entities: entities,
            entries: entries,
            group: group,
            official_announcement: official_announcement,
            legal_citations: extract_legal_citations(announcement)
          }
        end

        private

        def create_entity(entity_data)
          case entity_data.type
          when 'person'
            create_person_entity(entity_data)
          when 'organization'
            create_organization_entity(entity_data)
          when 'vessel'
            create_vessel_entity(entity_data)
          else
            create_organization_entity(entity_data)
          end
        end

        def create_person_entity(data)
          Ammitto::PersonEntity.new(
            id: generate_entity_id(data),
            entity_type: 'person',
            names: transform_names(data.name),
            gender: data.gender,
            birth_info: transform_birth_info(data.birth_info),
            nationalities: data.nationalities,
            # ... other fields
          )
        end

        def create_organization_entity(data)
          Ammitto::OrganizationEntity.new(
            id: generate_entity_id(data),
            entity_type: 'organization',
            names: transform_names(data.name),
            registration_number: data.registration_number,
            country: data.country,
            # ... other fields
          )
        end

        def create_entry(detail, entity_id)
          Ammitto::SanctionEntry.new(
            id: generate_entry_id(detail),
            entity_id: entity_id,
            authority: authority,
            regime: create_regime(detail.regime),
            effects: transform_effects(detail.measures),
            reasons: transform_reasons(detail.reasons),
            period: create_temporal_period(detail.period),
            status: 'active',
            announcement: create_announcement_reference(detail)
          )
        end

        # ... additional helper methods
      end
    end
  end
end
----

== Step 5: Create Rakefile for Data Generation

[source,ruby]
----
# Rakefile
# frozen_string_literal: true

require 'fileutils'

desc 'Clean generated API files'
task :clean do
  api_dir = File.expand_path('api', __dir__)
  # ... clean logic
end

desc 'Generate API files from source YAML data'
task :generate do
  ammitto_path = File.expand_path('../ammitto/lib', __dir__)
  $LOAD_PATH.unshift(ammitto_path) if Dir.exist?(ammitto_path)

  require 'ammitto'
  require 'ammitto/cli/harmonize_command'

  sources_dir = File.expand_path('..', __dir__)
  output_dir = File.expand_path('api', __dir__)

  options = {
    sources_dir: sources_dir,
    output_dir: output_dir,
    verbose: ENV['VERBOSE'] == 'true'
  }

  command = Ammitto::Cmd::HarmonizeCommand.new(options, [:new_source])
  command.run
end

desc 'Clean and regenerate all API files'
task regenerate: [:clean, :generate]

task default: :generate
----

== IRI Patterns

All entities use stable, list-agnostic IRIs:

```
Entity (LIST-AGNOSTIC):
  https://www.ammitto.org/entity/{source}/{local_id}
  Example: https://www.ammitto.org/entity/uk/12345

Entry (LIST-SPECIFIC):
  https://www.ammitto.org/entry/{source}/{list_type}/{local_id}
  Example: https://www.ammitto.org/entry/uk/consolidated-list/12345

Legal Instrument:
  https://www.ammitto.org/legal-instrument/{source}/{local_id}
  Example: https://www.ammitto.org/legal-instrument/uk/russia-regulation-2019

Announcement:
  https://www.ammitto.org/announcement/{source}/{document_id}

Group:
  https://www.ammitto.org/group/{source}/{document_id}
```

== Troubleshooting

=== Common Issues

1. **Transformer not found**: Ensure the transformer is registered in `Registry::TRANSFORMERS`

2. **Schema validation fails**: Check YAML structure matches schema definition

3. **Missing fields in output**: Ensure transformer maps all required fields

4. **Duplicate entities**: Use stable, list-agnostic entity IDs

5. **Language not displaying**: Ensure `lang` field is included in source data and exported

=== Debug Mode

Run generation with verbose output:

[source,bash]
----
VERBOSE=true bundle exec rake regenerate
----
