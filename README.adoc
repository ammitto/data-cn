= Ammitto China Sanctions Data Repository

== Purpose

This repository stores downloaded and processed sanctions data from the
People's Republic of China government sanctions announcements.

The data is maintained by the Ammitto project and used by the `ammitto` Ruby
gem for querying international sanctions information.

== Legal Basis

China maintains several sanctions and restricted entity lists:

* **不可靠实体清单 (Unreliable Entity List)** - Ministry of Commerce (MOFCOM)
* **反制裁清单 (Anti-Sanctions List)** - Ministry of Foreign Affairs (MFA)
* **出口管制管控名单 (Export Control List)** - MOFCOM

These are implemented through the Anti-Foreign Sanctions Law (2021) and
various ministerial regulations.

== Data Source

China maintains multiple sanctions lists:

.Unreliable Entity List (不可靠实体清单)
* **Authority**: Ministry of Commerce (商务部)
* **URL**: http://mofcom.gov.cn
* **Effects**: Import/export ban, investment ban

.Anti-Sanctions List (反制裁清单)
* **Authority**: Ministry of Foreign Affairs (外交部)
* **URL**: http://mfa.gov.cn
* **Effects**: Asset freeze, transaction ban, entry ban

.Export Control List (出口管制管控名单)
* **Authority**: Ministry of Commerce
* **Effects**: Export restrictions

=== Schema Types

Each YAML file follows one of three schema types:

. **Announcement Schema** - Standard sanction announcements with `sanction_details`
. **Modification Schema** - Temporal modifications with `measure_modifications`
. **Legal Instrument Schema** - Laws and regulations with structured content

Schema files are located in `schemas/`:

* `schemas/cn-announcement.yml` - For sanction list announcements
* `schemas/cn-measure-modification.yml` - For measure modifications (suspend/stop)
* `schemas/cn-legal-instrument.yml` - For legal instruments (laws, regulations)

=== Sanction Updates (Measure Modifications)

Measure modification files track temporal changes to sanctions:

[source,yaml]
----
# yaml-language-server: $schema=../../../schemas/cn-measure-modification.yml
---
announcement:
  title: 商务部新闻发言人就调整不可靠实体清单措施答记者问
  url: https://www.mofcom.gov.cn/...
  publish_date: 2025-05-14
  publish_time: "22:00"
  authority: 商务部新闻办公室
  content: |
    有记者问：...

measure_modifications:
  instruments:
    - law: 中华人民共和国反外国制裁法
    - law: 不可靠实体清单规定
  modifications:
    - action: suspend                      # or "stop"
      target_announcement_id: 〔2025〕7号
      target_announcement_date: 2025-04-04
      effective_date: 2025-05-14
      until_date: 2025-08-12               # Optional: when suspension ends
      notes: 暂停4月4日公告相关措施90天
----

Modification actions:

[cols="1,3"]
|===
| Action | Description

| `suspend`
| Temporarily suspend measures until a specified or unspecified date

| `stop`
| Permanently stop/terminate measures

|===

== Normalized Knowledge Graph Structure

The data follows a normalized structure where:

* **Entities are LIST-AGNOSTIC** - A person or organization can appear on multiple lists
* **Lists define sanctions regimes** - Each list has its own metadata
* **Entries are junction records** - They link entities to lists with list-specific data

=== Directory Structure

```
data-cn/processed/
├── entities/                      # NORMALIZED - unique per source
│   ├── mitsubishi-heavy-industries.yaml    # No prefix, just local ID
```

=== IRI Structure

All resources use normalized IRIs:

```
# Entity (list-agnostic)
https://www.ammitto.org/entity/cn/{local_id}

# Entry (list-specific)
https://www.ammitto.org/entry/cn/{list_type}/{local_id}

# List
https://www.ammitto.org/list/cn/{list_type}
```

Example IRIs:

```
https://www.ammitto.org/entity/cn/mitsubishi-heavy-industries
https://www.ammitto.org/entry/cn/import-export-control-list/mitsubishi-heavy-industries
https://www.ammitto.org/entry/cn/unreliable-entity-list/mitsubishi-heavy-industries
https://www.ammitto.org/list/cn/import-export-control-list
```

== Harmonized Data Format

After harmonization, entities are transformed into JSON-LD format using the
Ammitto gem's harmonized models.

=== Source YAML Schema

The source YAML files in `sources/` follow two schema types:

==== Announcement Schema (Type 1)

Standard sanction announcements with `sanction_details`:

[source,yaml]
----
announcement:
  title: 关于对...采取反制裁措施的决定
  url: https://www.mfa.gov.cn/...
  publish_date: 2022-12-23
  publish_time: "08:00"                  # Optional: HH:MM format
  authority: 中华人民共和国外交部
  publisher: 外交部                       # Optional
  type: 中华人民共和国外交部令
  document_id: 第4号
  signatory: 王毅                         # Optional
  signatory_title: 部长                   # Optional
  content: |                              # Full announcement text
    现公布《关于对...》

sanction_details:
  instruments:
    - law: 中华人民共和国反外国制裁法
      articles:                          # Optional
        - 第四条
        - 第五条
  entities:
    - name:
        zh-Hans: 余茂春
        en: Miles Yu                     # Optional
      type: individual                   # or organization
      effective_date: 2022-12-23
      effective_time: "08:00"            # Optional: HH:MM format
      sanction_list: 反制裁清单
      reason:
        - zh-Hans: 美方借口所谓...
          en: The US imposed...          # Optional
      measures:
        - type:
          - asset_freeze
          - prohibit_transactions
          zh-Hans: 冻结在我国境内的...
          en: Freeze all assets...       # Optional
      title:                             # Optional (for individuals)
        zh-Hans: 哈德逊研究所所长
        en: Director of Hudson Institute
      gender: male                       # Optional (for individuals)
----

==== Measure Modification Schema (Type 2)

Temporal modifications (suspend/continue/stop) with `measure_modifications`:

[source,yaml]
----
announcement:
  title: 商务部新闻发言人就调整不可靠实体清单措施答记者问
  publish_date: 2025-05-14
  publish_time: "22:00"
  authority: 商务部新闻办公室
  content: |

measure_modifications:
  instruments:
    - law: 中华人民共和国反外国制裁法
  modifications:
    - action: suspend                     # or continue_suspend, stop
      target_announcement_id: 〔2025〕7号
      target_announcement_date: 2025-04-04
      effective_date: 2025-05-14
      effective_time: "22:00"             # Optional: HH:MM format
      duration_days: 90                   # Optional
      duration_description: 1年           # Optional
      affected_entities: 11家美国实体
      notes: 暂停4月4日公告相关措施90天
----

=== Harmonized Ontology Models

The ammitto gem provides a comprehensive harmonized ontology that serves as a
superset of all sanctions data structures across different sources. The following
new models have been added to support China sanctions data:

==== LocalizedString

Represents text in a specific language with optional script and region metadata.

[source,yaml]
----
value: 三菱重工業株式会社
language: zh
script: Hani
region: CN
is_primary: true
is_transliteration: false
----

Fields:

[cols="1,2,2"]
|===
| Field | Type | Description

| `value`
| string
| The text content

| `language`
| string
| ISO 639-1 code (zh, en, ru, ar)

| `script`
| string
| ISO 15924 code (Latn, Hani, Cyrl, Arab, Hant)

| `region`
| string
| ISO 3166-1 code (CN, TW, HK)

| `is_primary`
| boolean
| Whether this is the primary representation

| `is_transliteration`
| boolean
| Whether this is a transliteration from another script

| `transliteration_system`
| string
| Transliteration system (pinyin, wade-giles, hepburn)

|===

  - 第四条
  - 第五条

quoted_text:
  - value: 为了维护国家主权...
    language: zh
----

Fields:

[cols="1,2,2"]
|===
| Field | Type | Description

| `id`
| string
| Unique IRI identifier

| `legal_instrument_id`
| string
| Reference to the LegalInstrument

| `articles`
| array[string]
| Specific articles cited (e.g., "第四条")

| `sections`
| array[string]
| Specific sections cited

| `paragraphs`
| array[string]
| Specific paragraphs cited

| `citation_type`
| string
| Type: legal_basis, reference, amendment, interpretation

| `context`
| string
| Why this instrument is cited

| `quoted_text`
| LocalizedString[]
| Quoted text from the instrument

|===

== Special Considerations

* **Format**: HTML announcements (not structured XML/JSON)
* **Language**: Chinese with English transliterations
* **Parsing**: Requires HTML extraction from official announcements
* **Encoding**: UTF-8 with Chinese characters
* **Multiple Lists**: China has 3 distinct sanctions lists - entities may appear on multiple

== Updating Data

Data is automatically updated daily via GitHub Actions.

To manually update using the ammitto gem:

[source,shell]
----
# From the ammitto gem directory
bundle exec ruby exe/ammitto fetch cn --format yaml --output-dir ./processed
----

To migrate existing data to normalized structure:

[source,shell]
----
# Run the migration script
ruby scripts/migrate_iri_structure.rb /path/to/data-cn/processed
----

== Knowledge Graph Integration

This repository's processed YAML data feeds into the Ammitto knowledge graph.

When the export process runs:

[source,shell]
----
# Export all knowledge graph data
ruby scripts/export_knowledge_graph.rb /path/to/repos /path/to/output
----

The cn data is transformed into:

* Individual JSON-LD nodes:
  - `api/v1/node/entity/cn/{id}.jsonld`
  - `api/v1/node/entry/cn/{list_type}/{id}.jsonld`
  - `api/v1/node/list/cn/{list_type}.jsonld`
* Lightweight search index entries with `listType` field
* List type facets for filtering
* By-list slices: `api/v1/by-list/cn/{list_type}.jsonld`
